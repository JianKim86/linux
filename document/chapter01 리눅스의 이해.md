#### 1.1 운영체제의 개요

운영체제(OS: Operating System)는 컴퓨터 하드웨어와 컴퓨터 사용자 간의 매개체 역할을 하는 시스템 소프트웨어로 사용자가 프로그램을 수행할 수 있는 환경을 제공한다.

- 좁은 의미 : 커널 (Kernel) 하드웨어와 응용 프로그램간 다리 역할

- 넓은 의미 : 커널, 미들웨어, 응용프로그램 실행 환경, 사용자 인터페이스 프레임워크
- 주목적: 편리하게 컴퓨터 시스템 이용
- 추가적 목적: 컴퓨터 하드웨어(CPU, 주기억장치, 입출력장치 등)의 효율적 관리

##### 1.1.1 운영체제의 이해

- **주요 역할**

  - 컴퓨터 하드웨어 제어
  - 작업 스케쥴링 및 입출력 연산 제어
  - 프로그램 실행 제어, 데이터 파일 저장 관리
  - 사용자의 하드웨어 자원 공유
  - 오류의 발생을 막고 복구 지원
  - 데이터의 조직화, 네트워크 통신 처리 기능
  - 편리한 UI 제공

- **주요 특징**

  - 다중 사용자 시스템 (Multi- User System)
  - 다중 작업 시스템 (Multi- Tasking System)
  - 강력한 네트워크 지원
  - UI제공
  - 계층적 파일 시스템 운영
  - 가상 메모리 지원
  - 고성능 프로세서에 최적화
  - 개방형 운영체제화
  - 뛰어난 이식성 지원
  - 가상화 기술 지원

- **운영체제의 역사** 

  초기전자식 디지털 컴퓨터(운영체제 x):1940 -> 단순 순차처리:1950->다중프로그래밍(시분할 처리):1960->다중모드 시분할(일괄처리,시분할 및 실시간 처리의 보편화):1970->분산네트워크(운영체제 기능들이 하드웨어에 포함된 펌웨어 개념 대두):1980-> 병렬 계산과 분할 계산(중앙집권식에서 분산으로,pc용 서버용 운영체제 보편화):1990->모바일 및 임베디드(개방형 시스템 발달):2000~

  

##### 1.1.2 운영체제의 종류

- **운영체제 분류**
  - 서버: 웹서버 ,메일 서버, DNS서버(도메인이름과 IP주소를 변환해주는)
  - 데스크톱
  - 모바일 및 임베디드: 휴대폰,PDA, 스마트폰,태블릿,자동차에 내장되는 IVI(In-Vehicle Infotainment)
  - 기타분야: 웹 기반 운영체제 (크롬OS)
- **주요 서버 및 데스크톱 운영체제**
  - 유닉스, 윈도우, Mac OS X
- **모바일 운영체제**
- **리눅스기반의 모바일 운영체제**
  - 안드로이드(구글),바다OS(삼성), 마에모(노키아), 모블린(인텔),미고(인텔+노키아),리모(삼성+파나소닉+...),타이젠(인텔+삼성+리눅스제단+...)
- **스마트 TV**
- **IVI(In-Vehicle Infotainment)**
  - 자동차 안 음악및 내비게이션 시스템등을 제공하는 하드웨어 장치의 모음
  - MS의 Windows Embedded Automotive ,QNX,GENIVI,안드로이드,MeeGo/Tizen IVI 

#### 1.2 리눅스의 역사와 철학

##### 1.2.1 리눅스의 탄색과 역사

리누스토발즈 가 미닉스(교육용 유닉스)의 기능을 만족하지 못해 새로운 운영체제를 개발 ->1991 첫 공개 ->GNU 프로젝트 -> 1990년 후반 레드햇, 칼데라 와 같은 리눅스배포판 등장 ->IVI에도 사용될만큼 다양한 분야로 확장

##### 1.2.2 리눅스 배포판

- **최초의 리눅스 배포판**
  - 소프트랜딩 리눅스 시스템(SLS) [1992년 피터 맥도널드] - 리눅스 커널 ,GNU프로그램, x 윈도 시스템, 기본적인 유틸리티 포함
  - 슬렉웨어[패트릭볼커딩] - SLS의 버그 개선
  - Yggdrasil Linux ,데비안 프로젝트[이안 머독] - SLS 다른 리눅스 배포판으로 대체
- **리눅스 배포판의 분류**
  - 슬렉웨어 - SuSE, Porteus, Vector Linux, Salix OS
  - 데비안 - Knoppix,Corel,Lindows,Ubuntu,Linux Mint, Elementray OS, 백트랙,칼리 : dpkg및 apt 패키지 관리도구 사용
  - 레드햇 - RHEL(Red hat Enterprise Linux), CentOS,Fedora, Oracle Linux, Scientific Linux, Asianux, Mandriva Linux, Mandrake : RPM 및  YUM 패키지 관리도구 사용
- **새로운 계파의 리더**
  - 수세(SUSE) -YaST 및 ZYpp패키지 관리 도구: 슬렉웨어에서 언어 수정 배포
  - 우분투 - 데비안의 패키지 관리기법 차용 (dpkg 및 apt) :데비안에서 사용자 편의성에 초점을 마춰  확장
- **주요 리눅스 배포판의 특징**
  - 슬렉웨어 - SLS를 기반으로 디자인의 안정성과 단순성을 목표, 내장되는 소프트웨어 패키지들은 최상단에서  최대한 수정되어 배포되는 관계, 사용자가 패키지 수정이 어려움/그래픽 기반의 설치절차 x , 소프트웨어 패키지의 의존성 자동으로 해결 x, 직접 수정 형태, 숙련된 사용자를 위한 리눅스
  - 데비안 - 자유소프트웨어로 구성된 유닉스 계열 운영체제, GNU GPL 라이세스 기반 데비안 프로젝트에서 최초버전 출시(1993), 정식판 지원 버전 GNU/Linux,슬렉웨어와 마찬가지로 SLS기반으로 만들어졌으나 차이점은 패키지 관리에 있다. 슬렉웨어에 비해 패키지 관리가 편리. 초기 dpkg 사용 최근에는 더욱 기능이 확장한 apt사용
  - 레드햇 - 그래픽 사용 인터페이스 제공,RPM(red hat package management)과 YUM(yellowdog updater,modified)라는 패키지 관리 도구 제공, Red Hat Linux 9까지만 소스공개및 무상 배포(2003), Red Hat Enterprise Linux(RHEL)는 유료배포판,추가적 무료배포판은 fedora라는 오픈 소스 프로젝트를 만들어 자유롭게 사용 가능하도록 함  
  - 수세 - Jurix 배포판을 기반으로 만들어짐
  - 우분투 - 그놈 데스크톱 환경 기반의 그래픽 셸인 유니티 사용: 사용자의 편의성에 초점
  - 기타 배포판- CentOS, 젠투리눅스, 리눅스 민트 등 300여 가지의 배포판 존재
  - 국내 리눅스 배포판 - 아시아눅스, SULinux, 안녕 리눅스 등

#####  1.2.3 리눅스 등장에 영향을 준 요소와 관련 용어

- GNU : GNU's Not Unix , GNU는 유닉스가 아니다. 유닉스를 모태로 더 강력한 운영체제를 만들고자 하는 프로젝트[리처드스콜먼], 소프트웨어의 상업화에 반대해 소프트웨어를 자유롭게 사용하도록 하는데 목적이 있음
- FSF(Free Software Foundation): 1985년 리처드 스콜먼이 설립한 자유 소프트웨어 재단(FSF), 자유 소프트웨어의 생산, 보급, 발전을 목표로 만들어진 비영리 조직, 
  - 자유소프트웨어의 4가지 자유
    - 목적에 상관없이 프로그램을 실행시킬 수 있는 자유
    - 프로그램이 어떻게 동작하는 지 학습하고, 필요에따라 프로그램을 개작할  수 있는 자유(이러한 자유가 실제로 보장되기 위해서는 소스코드를 이용할 수 있어야 한다.)
    - 무료 또는 유료로 프로그램을 재배포 할 수 있는 자유
    - 프로그램을 개선시킬 수 있는 자유와 개선된 이점을 공동체 전체가 누릴 수 있도록 발표할 수 있는 자유
- GNU GPL(General Public License) :저작권(Copyright)의 반대를 뜻하는 의미로 사용되는 Copyleft를 표기용어로 사용, GNU GPL은 GUN 소프트웨어에서 카피레프트를 실제로 구현한 라이선스를 뜻함, GNU정신에 입각하여 자유 소프트웨어를 보호하며 실제적으로 카피레프트를 담보할 수 있는 법률적 허가권
- 커널(Kernel): 운영체제의 핵심, 시스템이 부팅될 때 로드 ,주요역할은 시스템 하드웨어 제어
  - 주번호 : 커널에 매우 큰 변화가 생긴 경우
  - 부번호 : 기능상 작은 변화가 생긴 경우
  - 패치레벨 : 버그가 수정되고 새로운 기능의 계획이 추가된 경우
- 리눅스와 GNU/Linux : GNU 시스템과 리눅스가 결합함으로써 하나의 완성된 자유 운영체제가 만들어짐. 이 시스템을 GNU/Linux라고 부르는데 이는 리눅스를 시스템 커널로 채용한 GNU시스템을 지칭한다.

##### 1.2.4 리눅스의 철학

인터넷을 통해 최초 개발자가 중심이 되어 수많은 개발자들이 참여하에 개발이 이루어짐

##### 1.2.5 리눅스와 주요 라이선스

- GPL(General Public License)
  - 컴퓨터 프로그램은 어떤 목적으로든지 사용할 수 있다. 단 번으로 제한하는 행위는 할 수 없다
  - 커뮤터 프로그램의 실행 복사본은 언제나 프로그램의 소스 코드와 함께 판매하거나 소스코드를 무료로 배포해야한다
  - 컴퓨터 프로그램의 소스 코드를 용도에 따라 변경 할 수 있다
  - 변경된 컴퓨터 프로그램 역시 프로그램의 소스코드를 반드시 공개 배포해야한다.
  - 변경된 컴퓨터 프로그램 역시 반드시 똑같은 라이선스인 GPL 라이선스를 적용해야한다.
- LGPL(Library/Lesser General Public License) : GPL이 적용된 라이브러리는 자유 소프트웨어를 만들때 뿐만아니라 독점 소프트웨어에서도 사용 가능함, LGPL를 사용해서 개발한 뒤 GPL 라이센스로 변경은 가능하나 GPL을 임의로 LGPL로 변경 할 수 없다.
- BSD(Berkeley Softwere Distribution) 라이센스: 버클리의 캘리포니아 대학에서 배포하는 공개소프트웨어 라이선스 ,누구나 개작이 가능하고 수정한 것을 제한 없이 배포할 수 있다. 2차적 파생물에 대한 원시 코드의 비공개 허용, 상용 소프트웨어에서도 사용 가능
- 아파치(Apache) 라이센스 :개인 상업 목적으로 사용 가능,다만 재배포할 경우 아파치 출처를 밝혀야한다. Apache HTTP Server, Hadoop, Hbase, Tomcat
- MPL(Mozilla Public License) 라이센스: 모질라 재단에서 규정한 라이선스 BSD와 GPL라이선스의 혼합적 성격 
- MIT(Massachusetts Institute if Technology) 라이센스: 라이선스와 저작권 관련 명시 필요, 개작 가능 수정본 재배포시 소스코드 비공개 가능, X Window System, JQuery,Node.js,Rubyon Rails

#### 1.3 리눅스의 특징

##### 1.3.1 리눅스의 특징

- 다중 사용자 및 다중 처리 시스템:하나의 시스템에 다수의 사용자 동시접송하여 사용 가능. 각 접속자들이 다수의 응용프로그램을 실핼 할 수 있다.(Multi-user,Multitasking)
- 완전히 공개된 시스템
- 뛰어난 네트워크 환경: 이더넷,SLIP,PPP,ATM등 다양한 네트워크 환경 지원,TCP/IP,IPX,AppleTalk 등 대부분의 네트워크 프로토콜 지원
- 다양한 파일 시스템 지원
- 뛰어난 이식성 :약간의 어셈블리와 대부분의 C언어로 작성되어있어 C부분을 다시 컴파일 함으로써 쉽게 다른 시스템에 이식할 수 있다.
- 유연성과 확장성 :유닉스계승및 확장
- 뛰어난 안정성과 보안성 :상용 운영체제에 비해 빠르게 오류 수정과 보안 관련된 패치를 발표
- 우수한 가격대 성능비
- 다양한 응용프로그램의 제공 : 셸과 기본 명령어, X Window 시스템 , 편집기
- 다양한 배포판 존재

##### 1.3.2 리눅스의 단점

- 기술지원의 부족
- 특정 하드웨어에 대한 지원부족 
- 사용자의 숙련된 기술 요구

##### 1.3.3 리눅스의 기술적인 특징

- 계층적인 파일구조: 트리구조
- 장치의 파일화
- 가상 메모리 사용
- 동적 라이브러리 지원
- 가상 콘솔: 6개의 가상 콘솔 제공 [CTRL]+[ALT]+[F1~6]
- 파이프(pipe) : 여러 명령을 조합할 떄 사용, | 기호를 사용
- 리다이렉션(redirection) : 출력결과를 파일로 저장하거나 파일의 내용을 프로세스의 입력으로 사용하는 기법

##### 1.3.4 리눅스 디렉터리 종류와 특징

- **디렉터리(Directory)의 개요** : 파일을 보관하는 곳, 최상위에 해당하는 루트('/')를 중심으로 하위 디렉터리에 다수의 디렉터리가 존재하는 형태의 트리구조를 갖추어 계층적으로 관리된다.

- **디렉터리 종류와 특징**

  |    종류     | 특징                                                         |
  | :---------: | :----------------------------------------------------------- |
  |      /      | 최상위에 있는 디렉터리 , 루트 디렉터리                       |
  |    /bin     | binary의 약자, 실행파일들이 들어있고 명령어들이 있어 보통 PATH가 설정되었다. |
  |    /boot    | 부팅시 필요한 파일들이 들어있는 디렉터리, 부트로더로 사용되는 grub도 포함 |
  |    /dev     | 하드디스크,터미널등  물리적인 장치 등을 파일화하여 관리하는 디렉터리 |
  |    /etc     | 시스템 환경 설정 파일 및 부팅과 관련된 여러가지 스크립트 파일 포함 |
  |    /home    | 개인 사용자들이 사용하는 홈디렉터리                          |
  |    /lib     | 각종 라이브러리가 저장되어 있는 디렉터리 , 커널 모듈 포함    |
  | /lost+found | fsck로 파일 시스템을 복구할 떄 작업하는 디렉터리             |
  |    /mnt     | CD-ROM,플로피 디스크, 하드디스크 ,네트워크 파일 시스템 등을 마운트할 떄 포인터가 되는 디렉ㅌ리, /media,/net등을 추가 제공 |
  |    /misc    | 자동 마운트되는 프로그램인 autofs에 의해 사용됨              |
  |    /opt     | 응용프로그램들의 설치를 위해 사용되는 디렉터리               |
  |    /proc    | 가상파일 시스템,시스템의 프로세스의 상태정보, 하드웨어정보 기타 시스템 정보 포함 |
  |    /root    | 시스템 관리자인 root 사용자의 홈 디렉터리                    |
  |    /sbin    | 'System binary'시스템 관리에 대한 명령어들이 있다. 슈퍼유저인 root 가 사용, 시스템 종료, 네트워크 인터페이스 설정, 시스템 점검과 복구 등의 명령어들 포함 |
  |    /tmp     | 임시저장 디렉터리, 모든 사용자 접근 가능                     |
  |    /usr     | 시스템 운영에 필요한 명령, 응용프로그램들이 위치함 ,커널소스, c 언어 해더 파일,c-컴파일러 와 같은 개발도구 .X-window 아파치 my-sql, php 등이 이폴더 하위로 들어감 |
  |    /var     | 시스템 운영 로그파일, 스풀링과 같은 가변적 데이터 보관하는 디렉터리 |
  |   /media    | 이동식 장치인 cdrom, floppy등을 마운트하기 위해 제공하는 디렉터리, RHEL3번전후 /mnt에서 분리됨 |
  |  /selinux   | RHEL 4이후 등장 보안강화에 사용되는 디렉터리                 |
  |    /srv     | RHEL 4이후 등장, 사이트에서 생성되는 데이터를 저장하는 디렉터리 /var의 기능을 /srv로 대체될 가능 성이 있다. |
  |    /sys     | 2.6 커널에서 사용되는 가상 파일 시스템인 sysfs에서 사용하는 디렉터리 USB처럼 hot flug 하드웨어 정보를 가지고 있는 디렉터리 |
  |   /cgroup   | cgroup(control group)은 시스템 상에 동작 중인 테스크들을 임의로 그룹지어 제어 할 수 있도록 도와주는 기능을 제공하는데 그 관련 정보를 담고있는 디렉토리 |
  |   /lib64    | 64비트용 리눅스를 설치하는 경우 생성되는 64비트용 라이브러리 관련 디렉터리 |
  |    /run     | 부팅이후 동작중인 프로세스의 런타임 데이터를 저장 하고있는 디렉터리, RHEL 7 버전부터 등장 |

  

##### 1.3.5 리눅스 활용분야

- **리눅스 클러스터링(Linux Clustering)**

  클러스터 : 단말 제어 장치와 그에 접속된 복수 단말의 총칭 , 서버분야에서의 클러스터는 여러대의 컴퓨터를 연결하여 하나의 컴퓨터를 사용하는것 처럼 구성된 시스템을 말함

  - 고계산용 클러스터(HPC) : 고성능의 계산 능력을 제공하기 위한 슈퍼컴퓨터 구성에 주로 사용, 다른 말로 베어울프(beowulf) 클러스터, 병렬 처리용 슈퍼 컴퓨터의 개발을 시작하는 프로젝트에서 탄생
  - 부하분산 킅러스터(LVS-Linux Virtual Server) : 서버분야에서 사용, 리얼 서버에 부하를 분산해주는 로드벨런서를 두고 운영하는 방법
  - 고가용성 클러스터(HA) : 서버분야에서 사용, 지속적인 서비스 제공을 목적으로 부하분산 클러스터와 연동하여 사용됨.Backup Node(Secondary Node)가 Primary Node의 상태를 체크하고 있다가 이상이 발생하면 서비스를 이어받도록 구성

- **임베디드 시스템 (Embedded System)**

  - 임베디드 시스템 : 마이크로컨트롤러,마이크로프로세서,DSP(digital Singnal Processor) 등을 내장하여 특정 기능을 반복적으로 수행하기 위해 하드웨어와 소프트 웨어를 결합하여 만든 전자제어시스템, 고성능 소형 컴퓨터가 내장된 시스템
  - 임베디드 리눅스란 임베디드 컴퓨터 시스템에서 리눅스를 이용하는 것을 말한다.
  - 임베디드 리눅스의 장단점
    - 장점 
      - 별도의 로열티 및 라이선스 비용 x
      - 커널이 안정적
      - 관련 소프트웨어 개발 및 지원하는 업체가 많이 존재
      - 소스가 공개되어 변경하고 재배포하기 용의
    - 단점
      - 커널과 루트파일 시스템등에 상대적으로 많은 메모리를 차지
      - 사용자 모드와 커널 모드 메모리 접근이 복잡
      - 디바이스 드라이버 프레임워크가 복잡

- **클라우드 컴퓨팅(Cloud Computing)**

  - 클라우드 컴퓨팅의 정의 : 기존의 그리드 컴퓨팅, 분산 컴퓨팅, 유틸리티 컴퓨팅, 웹 서비스, 서버 및 스토리지 가상화 기술 소프트웨어등 기존의 기반 기술들을 융합하여 하나의 커다란 구름과 같은 컴퓨팅 환경을 만드는 기술, 인터넷을 통해 다양한 단말에서 웹 브라우저 등 필요한 응용소프트웨어를 구동하여 작업을 가능하게 하는 이용자 중심의 컴퓨팅 환경
  - 클라우드 컴퓨팅에서 제공하는 서비스: 
    - Iaas(Infrastructure as a Service)-업무처리에 필요한 서버,데스크톱 스토리지 같은 IT 하드웨어 자원을 클라우드 서비스에서 빌려쓰는 형태, 
    - PaaS(Platform as a Service)- 업무에 필요한 소프트웨어를 개발 할 수 있는 환경(platform)을 클라우드에서 빌려쓰는 형태
    - SaaS (Software as a Service)- 기업에서 사용하는 소프트 웨어를 통째로 클라우드 서비스 사업자에게 빌려쓰는 형태

- **빅 데이터(Big Data)**

  - 빅데이터 정의 : 대량의 정형 또는 비정형 데이터 집합  및  이러한 데이터로서 가치를 추출하고 결과를 분석하는 기술
  - 빅데이터의 관련 기술 : 데이터 마이닝(통계 학및 전산학),기계 학습, 자연언어 처리, 패턴 인식,NoSQL,R

- **사물인터넷(IOT: Internet of Things)**

  - 사물인터넷의 정의 :생활속 사물들을 유무선 네트워크로 연결해 정보를 공유하는 환경
  - 사물인터넷 플렛폼 : 사물간 인터넷을 할 수 있는 물적 기반인 통신 네트워크가 원활하게 작동하도록 하는 운영체제
